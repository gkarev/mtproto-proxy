# Implementation Plan: Critical Improvements

## Цели
Устранить критические уязвимости, повысить надёжность и улучшить UX на основе CODE_REVIEW.md.

---

## Фазы

### Фаза 1 — Безопасность и надёжность (P0)
1. Добавить централизованное логирование (`logging`)
2. Заменить generic `except:` на конкретные исключения с логированием
3. Добавить валидацию портов (диапазон 1024–65535, занятость)
4. Исправить hardcoded путь к MTG-бинарнику (env var)
5. Добавить проверку возвращаемого кода у `get_running_containers()`
6. Добавить try/except в `save_config()`

### Фаза 2 — UX/UI (P1)
7. Добавить подтверждение перед удалением прокси
8. Улучшить подписи кнопок управления (добавить текст к иконкам)
9. Статус-зависимые кнопки в меню управления (показывать только актуальные)
10. Показывать секрет в деталях прокси (кнопка "Показать")

### Фаза 3 — Качество кода (P2)
11. Убрать дублирование `show_proxy_status` / `show_proxy_status_command`
12. DRY: вынести сборку клавиатуры управления прокси в отдельную функцию
13. Удалить дублирующийся `import subprocess` внутри функции
14. Убрать неиспользуемые CommandHandler'ы (`/proxy_list`, `/proxy_status`)

---

## Ожидаемый результат
- Все P0 проблемы устранены
- UX: нет безвозвратных действий без подтверждения
- Код: нет silent failure, все ошибки логируются
- README обновлён
